# 🚀 [PRD] 프로젝트명: 퍼즐 나이트 (Puzzle Knight)

## 1. 프로젝트 개요
- **장르:** 하이브리드 캐주얼 (3매치 퍼즐 + 방치형 RPG)
- **플랫폼:** 모바일 (Android 우선, iOS 대응)
- **목표:** 1년 내 MVP 개발 및 글로벌 런칭 (Habby형 수익 모델 적용)
- **핵심 가치:** - 퍼즐 매칭을 통한 실시간 스킬 발동의 타격감.
  - 방치형 시스템을 통한 지속적인 성장 피드백.
  - 다중 히어로 덱 빌딩을 통한 전략적 조합 및 레이드 확장성.

---

## 2. 게임플레이 메커니즘 (In-Game)

### 2.1 화면 및 보드 구성
- **화면 비:** 9:16 (세로형 고정)
- **퍼즐 보드 규격:** **7(가로) x 6(세로)** (총 42칸)
- **레이아웃:**
  - [상단 40%] 전투 뷰: 아군 히어로 파티(최대 5명, 좌측) vs 다수 몬스터 웨이브(우측) 자동 전투 연출
  - [중단 10%] 상태창: 히어로별 HP 바, 개별 궁극기 게이지(초상화 테두리), 콤보 카운트 UI
  - [하단 50%] 7x6 퍼즐 보드

### 2.2 히어로 덱 및 블록-색상 매핑 시스템
히어로의 배치 인덱스(Index)와 퍼즐 블록 색상을 1:1로 매핑한다. 최소 3명, 최대 5명 출전.
배치된 히어로의 블록 색상만 보드에 드롭되며, 히어로 사망 시 해당 색상 블록은 즉시 제거되고 드롭이 중단된다.

| 배열 인덱스 | 위치 | 블록 색상 | 비고 |
| :--- | :--- | :--- | :--- |
| **Index 0** | 1열 (최전방) | 🔴 빨강 | 필수 (최소 인원) |
| **Index 1** | 2열 | 🟡 노랑 | 필수 (최소 인원) |
| **Index 2** | 3열 | 🟢 초록 | 필수 (최소 인원) |
| **Index 3** | 4열 | 🔵 파랑 | 4인 이상 배치 시 활성 |
| **Index 4** | 5열 | 🟣 보라 | 5인 배치 시 활성 |

매칭된 블록의 색상에 대응하는 히어로의 고유 스킬이 즉시 발동된다.

### 2.3 핵심 전투 로직
- **개별 스킬 발동:** 연쇄(Cascade)로 터지는 블록마다 해당 색상에 매핑된 히어로의 스킬이 순차적으로 발동된다. (v1의 '첫 매치 색상 통일' 규칙 폐기)
- **콤보 시스템:** 블록이 터질 때마다 `Combo +1`. 증폭 공식: `최종 효과량 = 기본 효과량 * (1 + (현재 콤보 - 1) * 0.2)`. 콤보 캡 없이 무제한 증폭.
- **궁극기(Ultimate):** 히어로별 개별 게이지. 자기 색상 블록 매칭 시에만 해당 히어로 게이지 충전. 100% 도달 시 초상화 점등 -> 유저가 직접 탭(Tap)하여 해당 히어로의 필살기 발동.
- **타겟팅:** 아군 공격은 가장 전방의 적을 우선 타격 (유저 터치로 일점사 타겟 지정 가능). 적군 공격은 아군 Index 0 히어로를 우선 타격 (도발 스킬이 없을 경우).

## 3. 메타 게임 및 비즈니스 모델 (Habby Style)

### 3.1 로비 및 내비게이션
- **구조:** 하단 4개 탭 스와이프 인터페이스 (상점 / 성장 / 전투 / 던전)
- **성장:** 히어로 레벨업(골드) 및 블록 스킬 강화(룬 재화) 중심. (장비 시스템 제외)

### 3.2 수익 모델 (BM)
1. **에너지 시스템:** 스테이지 진입 시 '번개' 소모. (10분당 1개 회복, 서버 검증)
2. **보상형 광고:** 에너지 즉시 충전, 오프라인 방치 보상 2배 수령.
3. **인앱 결제:** - 광고 제거 패키지 (영구 효과)
   - 프리미엄 배틀 패스 (시즌별 보상)
   - 성장 가속 패키지

## 4. 기술 아키텍처 및 확장성 (Server/Client)

### 4.1 서버 (C# .NET Core)
- **검증 로직:** 전투 결과(로그) 및 에너지 회복 시간을 서버 측에서 UTC 기준으로 검증하여 치트 방지.
- **데이터 관리:** 유저 정보, 스테이지 진행도, 재화 보유량을 RESTful API 및 Redis(랭킹/세션)로 관리.

### 4.2 멀티플레이(레이드) 확장 설계
- **통신:** 레이드 도입 시 실시간 데이터 전송을 위해 `Protobuf` 기반의 바이너리 소켓 통신(SuperSocket/gRPC) 사용.
- **구조:** 레이드 참여 시 각 플레이어의 '스킬 발동 패킷'을 공유하여 보스의 체력을 동시 삭감하는 '비동기적 실시간' 방식 지향.

## 5. 개발 마일스톤
- **1단계 (프로토타입):** 7x6 보드 로직 및 콤보/스킬 연동.
- **2단계 (서버 연동):** 회원가입, 방치형 보상, 에너지 시스템 API 구축.
- **3단계 (폴리싱):** 하단 탭 UI, 광고 SDK 및 결제 모듈 연동.
- **4단계 (테스트):** 밸런스 QA 및 소프트 런칭.
