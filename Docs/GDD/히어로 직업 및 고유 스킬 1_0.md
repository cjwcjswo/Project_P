# 🚀 [GDD] 히어로 직업 및 고유 스킬 시스템 (v3.0)

## 1. 직업군 (Class) 시스템
히어로는 4가지 특화 직업을 가지며, 직업에 따라 성장 스탯의 가중치가 다르게 부여된다. 유저가 퍼즐판에서 직관적으로 직업을 식별할 수 있도록 `BlockView` 중앙에 직업 아이콘을 오버레이(Overlay)하여 렌더링한다.

| 직업명 | 역할군 (Role) | 주력 스탯 | 블록 아이콘 (UI) | 추천 스킬 타입 (참고사항) |
| :--- | :--- | :--- | :--- | :--- |
| **전사 (Warrior)** | 파티의 탱커 | HP, Defense | 🛡️ 방패 | 도발, 방어력 증가, 단일 스턴 |
| **마법사 (Mage)** | 광역 딜러 및 메저 | Attack | 🪄 지팡이/별 | 광역 공격(All), 상태 이상 |
| **암살자 (Assassin)** | 단일 극딜러 | Attack (극대화) | 🗡️ 단검 | 강력한 단일 공격, 방어력 무시 |
| **힐러 (Healer)** | 파티 서포터 | HP, Defense | ➕ 십자가/하트 | 회복(Heal), 방어막(Shield), 버프 |

## 2. 영웅 등급(Grade)별 스킬 트리거 시스템
영웅의 태생 등급(1성 ~ 3성)에 따라 보유할 수 있는 스킬의 개수와 발동 방식이 확장된다. 

### 🌟 1성 (Grade 1): 기본 퍼즐 영웅
* **발동 조건:** 퍼즐 보드에서 해당 영웅과 매핑된 색상의 블록을 매칭(3개 이상) 시 발동.
* **보유 스킬:** `MatchSkillId` 1개 보유.

### 🌟🌟 2성 (Grade 2): 특수 블록 생성 영웅
* **특수 블록 생성:** 해당 색상의 블록을 **4개 이상** 매칭하면, 매칭된 중심 위치에 '히어로 고유 스킬 블록'이 1개 생성된다.
* **고유 스킬 블록 시각화:** 기존 `BlockView`에 반짝이는 파티클/아우라 이펙트(`GameObject.SetActive`)를 켜서 일반 블록과 확실하게 구분한다.
* **발동 조건:** 생성된 특수 블록을 스왑(이동)하지 않고 **터치(Tap)** 시 즉시 발동.
* **발동 효과:** 1. 보드판의 상하좌우(십자 모양) 블록 파괴 로직 실행.
  2. 영웅 데이터에 지정된 `UniqueSkillId` 즉시 발동.

### 🌟🌟🌟 3성 (Grade 3): 궁극기(Ultimate) 영웅
* 2성의 모든 기능(특수 블록 생성)을 포함한다.
* **초상화 궁극기 추가:** * 매칭 및 콤보가 발생할 때마다 해당 영웅의 초상화 게이지가 충전된다.
  * 100% 충전 시 유저가 **초상화를 터치(Tap)**하여 발동.
* **발동 연출 (Cut-in):** 궁극기 발동 시, 투명도가 적용된 영웅의 고해상도 일러스트가 화면 좌측에서 우측으로 자연스럽게 스쳐 지나가는(Fade in-out) 컷인 이펙트를 재생한다.
* **보유 스킬:** `MatchSkillId`, `UniqueSkillId`, `UltimateSkillId` 총 3개 보유.

## 3. 데이터 명세 변경 (`HeroData.json`)
기존의 `SkillId` 단일 속성을 등급별 스킬 구조에 맞춰 확장한다. 
*하위 등급의 경우 사용하지 않는 스킬 ID는 `0`으로 처리하여 예외 방지.*

```json
[
  {
    "Id": 10001,
    "DisplayName": "견습 기사 아서",
    "PrefabPath": "Heroes/Knight_Arthur",
    "IllustrationPath": "Illustrations/Knight_Arthur_Cutin",
    "Class": "Warrior",
    "Grade": 3,
    "MatchSkillId": 5001,     // 3매칭 시 발동 (기본 스킬)
    "UniqueSkillId": 5005,    // 4매칭 특수 블록 터치 시 발동 (고유 스킬)
    "UltimateSkillId": 5010,  // 초상화 터치 시 발동 (궁극기)
    "BaseStats": { "MaxHP": 1200, "Attack": 80, "Defense": 30 },
    "GrowthStats": { "MaxHP": 250, "Attack": 10, "Defense": 8 },
    "AutoAttackInterval": 1.5,
    "AutoAttackRatio": 0.1
  },
  {
    "Id": 10004,
    "DisplayName": "전사 크롬",
    "PrefabPath": "Heroes/Warrior_Crom",
    "IllustrationPath": "",
    "Class": "Warrior",
    "Grade": 1,
    "MatchSkillId": 5001,     // 1성은 일반 매칭 스킬만 보유
    "UniqueSkillId": 0,       // 없음
    "UltimateSkillId": 0,     // 없음
    "BaseStats": { "MaxHP":  900, "Attack": 120, "Defense": 18 },
    "GrowthStats": { "MaxHP":  180, "Attack":  18, "Defense":  4 },
    "AutoAttackInterval": 1.8,
    "AutoAttackRatio": 0.12
  }
]
```
*(참고: `SkillData.json`은 이전 기획의 ActionType, TargetScope 기반 확장형 구조를 그대로 유지하며, 각 영웅의 ID에 맞게 연결(Mapping)만 수행한다.)*
