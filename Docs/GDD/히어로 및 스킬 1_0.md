# 🚀 [GDD] 히어로 및 스킬 시스템 (Data-Driven)

## 1. 개요
히어로와 스킬 데이터는 소스 코드와 분리되어 독립적인 JSON 스키마로 관리된다. 이를 통해 클라이언트/서버 빌드 없이 신규 영웅과 스킬을 추가할 수 있다. 스킬 시스템은 '타입(ActionType)'과 '범위(TargetScope)', 그리고 '상태 이상(StatusEffects)'을 조합하는 방식으로 설계하여 확장성을 극대화한다.

## 관련 문서
- [`스킬 시스템 고도화 1_0.md`](스킬%20시스템%20고도화%201_0.md): Actions[] + Target 객체 기반의 모듈식 스킬 구조(확장안)
- [`히어로 직업 및 고유 스킬 1_0.md`](히어로%20직업%20및%20고유%20스킬%201_0.md): 직업(Class), 등급별 스킬 트리거(고유 스킬 블록/궁극기)

## 2. 히어로 (Hero) 시스템

### 2.1 히어로 데이터 명세 (`HeroData.json`)
* **Id (`uint`)**: 히어로 고유 식별자.
* **DisplayName (`string`)**: 노출되는 영웅 이름.
* **PrefabPath (`string`)**: 3D/2D 모델 및 애니메이션 리소스 경로.
* **Grade (`int`)**: 태생 등급 (1 ~ 3성).
* **SkillId (`uint`)**: 보유하고 있는 고유 스킬 ID (`SkillData.json` 매핑).
* **BaseStats**: 1레벨 기준 기본 스탯 (`MaxHP`, `Attack`, `Defense`).
* **GrowthStats**: 레벨업 시 1레벨당 증가하는 스탯 수치.
* **AutoAttackInterval (`float`)**: 평타 공격 간격(초).
* **AutoAttackRatio (`float`)**: 평타 공격 시 적용되는 공격력 계수.

### 2.2 레벨 및 경험치 성장 공식
* **레벨 구조:** 초기 만렙 50Lv (추후 확장 가능).
* **스탯 성장 공식:** * `현재 스탯 = BaseStat + (Level - 1) * GrowthStat`
* **요구 경험치(EXP) 공식:** * `다음 레벨업 필요 EXP = 100 * (Level)^2 * 1.1^(Level/10)`

### 2.3 히어로 JSON 예시
```json
[
  {
    "Id": 10001,
    "DisplayName": "견습 기사 아서",
    "PrefabPath": "Heroes/Knight_Arthur",
    "Grade": 3,
    "SkillId": 5001,
    "BaseStats": { "MaxHP": 1000, "Attack": 100, "Defense": 20 },
    "GrowthStats": { "MaxHP": 200, "Attack": 15, "Defense": 5 },
    "AutoAttackInterval": 1.5,
    "AutoAttackRatio": 0.1
  }
]
```

## 3. 스킬 (Skill) 시스템

### 3.1 스킬 데이터 명세 (`SkillData.json`)
* **Id (`uint`)**: 스킬 고유 식별자.
* **DisplayName (`string`)**: 스킬 이름.
* **EffectPrefabPath (`string`)**: 스킬 발동 시 출력할 이펙트/VFX 경로.
* **ActionType (`string`)**: 스킬의 주 목적 (`Attack`, `Heal`, `Shield`, `Buff`).
* **TargetScope (`string`)**: 스킬 적용 대상 범위 (`Single`, `Multi`, `All`).
* **MaxTargetCount (`int`)**: `TargetScope`가 `Multi`일 때 타격할 최대 대상 수.
* **BaseMultiplier (`float`)**: 주 스탯(공격력 등)에 곱해질 스킬 계수.
* **StatusEffects (`Array`)**: 스킬 적중 시 부여할 부가 효과 리스트.

### 3.2 상태 이상 (Status Effects) 명세
* **EffectType (`string`)**: 효과 종류 (`Stun`, `Burn`, `AtkUp`, `DefDown` 등).
* **Value (`float`)**: 효과의 강도 (예: 방어력 20% 감소 시 0.2).
* **Duration (`float`)**: 유지 시간 (초).
* **Probability (`float`)**: 발동 확률 (0.0 ~ 1.0).

### 3.3 스킬 JSON 예시
```json
[
  {
    "Id": 5001,
    "DisplayName": "회전 베기",
    "EffectPrefabPath": "Effects/Skills/SpinSlash",
    "ActionType": "Attack",
    "TargetScope": "Multi",
    "MaxTargetCount": 3,
    "BaseMultiplier": 1.5,
    "StatusEffects": [
      {
        "EffectType": "Stun",
        "Value": 0.0, 
        "Duration": 1.5,
        "Probability": 0.3
      },
      {
        "EffectType": "DefDown",
        "Value": 0.2, 
        "Duration": 5.0,
        "Probability": 1.0
      }
    ]
  }
]
```
자세한 내용은 '스킬 시스템 고도화' 문서를 참고합니다.
