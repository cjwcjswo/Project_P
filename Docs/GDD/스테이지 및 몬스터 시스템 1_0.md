# 🚀 [GDD] 스테이지 및 몬스터 웨이브 시스템 (v1.0)

## 1. 개요
본 시스템은 게임의 스테이지 구성, 난이도 밸런싱, 보상 체계를 데이터 주도(Data-Driven) 방식으로 관리한다. 모든 스테이지와 몬스터 정보는 JSON 파일을 통해 동적으로 로드되며, 코드 수정 없이 새로운 콘텐츠를 추가할 수 있는 구조를 지향한다.

### 2.2 스테이지 구성 데이터 (`StageData.json`)
웨이브 구성과 스테이지 보상, 입장 비용을 정의한다.

```json
[
  {
    "StageId": 1,
    "StageName": "고블린 숲 1-1",
    "EntryCost": 5,
    "Waves": [
      {
        "MonsterIds": [1001, 1001]

      {
        "MonsterIds": [1001, 1001, 2005]
      }
    ],
    "ClearRewards": {
      "TotalExp": 1200,
      "TotalGold": 300
    }
  }
]
```

## 3. 핵심 시스템 로직

### 3.1 몬스터 스폰 및 배치
- **동적 스폰:** `StageData`의 `MonsterIds` 리스트를 순회하며 `MonsterData`에서 정보를 찾아 인게임에 생성한다.
- **인덱스 우선순위:** 리스트의 **0번 인덱스**에 해당하는 몬스터가 전투 진형의 가장 전방(탱커 위치)에 스폰된다.
- **외형 로드:** `PrefabPath`에 지정된 경로를 통해 유니티 프리팹을 로드하여 `EnemyState`와 연결한다.

### 3.2 보상 및 에너지 시스템
- **에너지 소모:** 스테이지 진입 시 즉시 `EntryCost`만큼 에너지를 소모한다.
- **경험치 분배 (N분의 1):** 스테이지 클리어 시 획득하는 `TotalExp`를 생존 여부와 관계없이 **출전한 히어로 수**로 나누어 배분한다.
  - 공식: `개별 획득 EXP = TotalExp / 출전 히어로 수` (소수점 내림 처리)
- **골드 획득:** 계정 귀속 재화로 즉시 합산된다.

---

### 4. 확장성 고려 사항
- **보상 시스템 확장:** 추후 아이템 드랍 리스트(DropTableId)를 `ClearRewards`에 추가할 수 있도록 설계한다.
- **몬스터 스킬:** `EnemyState.cs`에 정의된 쿨타임과 캐스트 시간을 기반으로 하되, 추후 JSON에 스킬 타입(광역, 단일, 디버프 등)을 추가 정의한다.
